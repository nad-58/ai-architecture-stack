# Glossary

## AI Architecture Stack

A layered framework for understanding modern AI systems from physical resources to applications and governance.

## Application Layer

The user-facing layer where AI capabilities become useful products, tools, dashboards, copilots, or integrated workflows.

## Orchestration

The coordination of prompts, tools, memory, routing, workflows, and multi-step task execution.

## Agent

A system component that can plan or execute a sequence of actions using model outputs, tools, rules, or workflow logic.

## Model Layer

The layer where models are selected, trained, fine-tuned, evaluated, optimised, and monitored.

## Data and Knowledge Layer

The layer responsible for data sources, data pipelines, retrieval, grounding, embeddings, vector databases, and knowledge access.

## RAG

Retrieval-Augmented Generation. A design pattern where a model retrieves relevant information before generating an answer.

## KAG

Knowledge-Augmented Generation. A broader pattern where structured or semi-structured knowledge supports model outputs.

## LLM

Large Language Model. A model designed to process and generate language and related representations.

## SLM

Small Language Model. A smaller model that may be easier to deploy locally, privately, or on constrained hardware.

## Edge AI

AI that runs close to a device, sensor, machine, or local environment instead of relying entirely on a remote cloud service.

## On-premise deployment

Deployment on infrastructure controlled by an organisation rather than a public cloud provider.

## Governance

The set of processes, controls, documentation, monitoring, and review activities used to ensure the AI system is suitable, reliable, and responsibly deployed.
