# Model-Family Boundaries: LLM, Large Vision Model, VLM, and VLA

Clear AI architecture begins by identifying the model family correctly.

## Family comparison

| Family | Inputs | Outputs | Primary role |
|---|---|---|---|
| Large language model | Text or tokens | Text, code, structured tokens | Language modelling |
| Large vision model | Images or video | Features, labels, boxes, masks, depth | Visual perception |
| Vision-language model | Vision plus language | Captions, answers, retrieval, grounding | Cross-modal understanding |
| Vision-language-action model | Vision, language, state or history | Actions, trajectories, policies | Embodied decision and control |

The acronym **LVM** is ambiguous. It can mean “large vision model” or “large vision-language model.” This repository writes the full term and uses **VLM** for vision-language model.

## Placement in the architecture stack

### Layer 5: Model and Learning

This layer contains the main model components:

- language backbone;
- vision encoder or vision foundation model;
- multimodal projector, cross-attention, or fusion module;
- action decoder or learned policy.

### Layer 4: Data and Knowledge

The data requirements differ:

- LLM: text, code, instructions, and preference data;
- large vision model: images, video, labels, masks, and spatial annotations;
- VLM: aligned image-text or video-text data;
- VLA: vision-language-action demonstrations, trajectories, and state data.

### Layer 6: Orchestration and Agents

A model does not become a VLA simply because an external agent calls a tool. Distinguish:

- an LLM or VLM orchestrating software tools;
- a learned VLA policy producing action outputs;
- a hybrid design where a VLM plans and another component executes.

### Layers 1 to 3: Physical, chips, and compute

Vision increases data bandwidth and accelerator demand. Video adds temporal and memory constraints. VLA systems add real-time timing, sensor synchronisation, and safe-state requirements.

### Layer 7: Application and Experience

- LLM: text interaction;
- large vision model: perception or visual-analysis output;
- VLM: multimodal answer, caption, retrieval, or grounding;
- VLA: action in software or a physical environment.

## System-boundary examples

### Image classifier

```text
Image → large vision model → class label
```

This is not a VLM because no language representation is used in the decision path.

### CLIP-style retrieval

```text
Image encoder + text encoder → shared embedding space → similarity score
```

This is a VLM because image and text representations are explicitly aligned, even though it does not generate free-form text.

### Multimodal assistant

```text
Image → vision encoder → fusion → language model → text answer
```

This is a generative VLM.

### Robot policy

```text
Camera + instruction + system state → VLA policy → action trajectory
```

This is a VLA because the model produces actions.

### VLM planner with separate controller

```text
Image + instruction → VLM plan → controller → command
```

The full application is an embodied AI system, but the VLM component is not automatically a VLA. The controller and orchestration layers should be documented and tested separately.

## Governance and testing consequences

Architecture documentation should record:

1. the actual model family;
2. each component’s inputs and outputs;
3. where modality fusion occurs;
4. whether actions are generated by the model or another component;
5. the human-override and safe-state design;
6. which architecture layer owns each control and test.
