# Governance by Layer

Governance should be considered across the full architecture, not only at the end of a project.

| Layer | Governance focus |
|---:|---|
| 7 | user communication, feedback, and appropriate use |
| 6 | tool permissions, workflow logging, review steps, and escalation |
| 5 | model evaluation, robustness testing, fairness checks, and versioning |
| 4 | data quality, provenance, privacy, and retrieval traceability |
| 3 | access control, monitoring, resilience, and deployment records |
| 2 | hardware suitability, performance limits, and device constraints |
| 1 | power, cooling, availability, and continuity |

## Cost, quality, speed, and safety

Every AI project balances quality, speed, cost, and safety. The best architecture makes these trade-offs explicit.

## Useful evidence

Useful evidence may include architecture diagrams, data lineage, model documentation, evaluation reports, test records, deployment records, monitoring logs, risk assessments, and user feedback summaries.
