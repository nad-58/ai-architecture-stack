# Contributing

Contributions that improve architecture explanations, diagrams, case studies, examples, and references are welcome.

## Guidelines

- Keep changes focused and technically justified.
- Preserve the layered architecture terminology used by the repository.
- Explain any proposed change to layer boundaries or responsibilities.
- Use original diagrams and publicly shareable references.
- Check Markdown links and SVG rendering before submitting.

## Commit prefixes

Use `feat:`, `fix:`, `docs:`, `test:`, `refactor:`, or `chore:`.

## Pull requests

Describe the purpose, files changed, architecture impact, and validation completed.
